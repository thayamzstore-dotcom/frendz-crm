Eventos Frendz — gatilhos de automação

Ative os eventos que devem disparar os funis de recuperação. A Frendz enviará os dados via postback para seu webhook configurado.

Aguardando pagamento
payment_status: waiting — PIX ou boleto emitido, pagamento não confirmado
waiting
Recusado
payment_status: refused — cartão recusado, refuse_reason preenchido
refused
Em processamento
payment_status: processing — enviado ao gateway, aguardando retorno
processing
Cancelado
payment_status: cancelled — transação cancelada antes da confirmação
cancelled

Como funciona: A Frendz detecta esses eventos via polling da API /public/v1/transactions com filtro de payment_status e payment_method. Configure o campo postback_url na request de pagamento para receber atualizações em tempo real.
